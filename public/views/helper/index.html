What is fake server local?

fake server local is a local REST api server which return random lorem-ipsum data based on user schema. It might be helpful for Front-end/App developers, who don't have sufficient data (might be REST api is not ready or not enough data in database) to test their projects. All you need, just create an api endpoint and output schema. You will get your sufficient data to test your project.

Technologies used: Node.js, Koa.js, neDB, JS. It take zero external configuration. So it also works without internet

How is it works?

Suppose you have a REST api server (for example http://api.example.com) and you have an api end-point like /some/api. You are expecting a JSON object for that api and ofcourse you know expected JSON object schema. Object value might be contained text | boolean | integer | textarea | array | object-array etc. Using fake server local, all you need just replace your REST api endpint and add expected output schema. You will get lorem-ipsum data based on your expected schema.

Setup

Step 1. First clone project from github

Step 2. Run npm install (node.js required) to install dependency.

Step 3. Run npm start

Step 4. Now go to http://localhost:9920/dashboard. You should see the following image.

Step 5. Create a new project. Now your api endpoint will be http://localhost:9920/${your-project-title}.

Initially you have to create a project. Why? Sometimes you have to create same api endpoint for multiple projects /login, /signup etc. and you are expecting different response data based on project. You can get different response data for same api endpont based on your project.

Create fake api

Assume you have already created project. If yes, then your api server will be http://localhost:9920/your-project-name. Now you can create your first fake api. First select Route type from dropdown. Route type can be GET | POST | PUT | PATCH | DELETE. Then add your route name, for example /some/api/name. You might be noticed, there is no save button. So if you select Route type and add route name, it will be saved and shown on left navbar on cursor moved. Now add returned schema. It has to be a JSON object as like as shown on image. Here object-key-* are user's object key and return-Type can be id | uuid | boolean | text | title | textarea | integer | float | phone | zipcode | date | time | date-time | url | email | image | pdf | csv | doc | ipaddress. image | pdf | csv | doc value will be some fixed default files which are attached with this project. title value will be 2-5 words. I believe, all other return-Type will be return data as it is named.

Assume you add a route name /some/api/name. Then your api end-point will be like http://localhost:9920/your-project-name/some/api/name. You have select route type GET and your expected returned JSON schema as like as shown on image right.

Now if you call GET request to http://localhost:9920/your-project-name/some/api/name, you will see the JSON data as like as shown on image right. (return random lorem-ipsum data)

More details

I hope we all have understand how this project work and what can we do. Here are some more details

We might have expected fixed value. For example, in json response, there might be a parameter, called message, that value will be always like message: "Successful". In this situation, simply add like { "object-key-name": "any-fixed-value" } if return-Type doesn't match with return-Type value, then expected value will be returned typed value.

We might have expected random value from fixed value list. For example, For example, in json response, there might be a parameter, called status hat value will be always be like status: "complete" or status: "failed". In this situation, simply add like { "object-key-name": "fixed-value-01,fixed-value-02" }. That means, if return-Type's value is comma-separated, then returned value will be one of comma-separated value.

We might have expected random array value. To get array value, try "object-key-name": ["return-Type"] (only one value and have to matched with return-Type). , see to bellow image.

In first image, It will return an array of ["return-Type"] and returned array contains about 1-10 results.

We create a special object, which contains "__type": "array" (means it return array value), "__range": "minRange,maxRange" return array data between __range. __range can also be fixed number, which means, it return fixed array length. "__property": "return-Type | object" array value will be based on __property. __property can be any return-Type or object

In third image, we add multiple array values, it will return a array, which contains one of array values.

We might want to expect random array object values. To get array object value, see to bellow image. In first image, we create an object array, and it will return 1-10 object array random values.

But sometimes, we want more data. In second image, we special object, which contains "__type": "array" (means it return array value), "__range": "minRange,maxRange" return array data between __range. __range can also be fixed number, which means, it return fixed array length. "__property": "return-Type | object" array value will be based on __property. __property can be any return-Type or object

We also need token data. see right images. Here we create a special object where __type is token and it required __property, __property can be object, array or return-Type

In POST | PUT | PATCH type api, if request.body's parameter match with user schema parameter, then request.body data will be returned
