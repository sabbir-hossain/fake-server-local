What is fake server local?

fake server local is a local REST api server which return random lorem-ipsum data based on user schema. It might be helpful for Front-end/App developers or developers who are building microservices, who don't have sufficient data (might be REST api is not ready or not enough data in database) to test their projects. All you need, just create an api endpoint and output schema. You will get your sufficient data to test your project.

Technologies used: Node.js, Koa.js, neDB, JS. It take zero external configuration. So it also works without internet

How is it works?

Assume you have an e-commerce app and it's REST Api endpoint is https://api.awesome-ecommerce.com.

Assume you get list of products by calling https://api.awesome-ecommerce.com/products/list and you are getting an object array that contains id and name. So your output schema is [{ id: 'number', name: 'text' }].

By using fake server local, you can create mock server that can send fake data based on your schema.

Now you can create a mock server. For example http://localhost:9920/e-commerce. Now your list of products api will be like http://localhost:9920/e-commerce/products/list and your output response schema will be [{ id: 'number', name: 'text' }]. You will get a fake response for calling your fake api. 

  
// response from 
// "https://api.awesome-ecommerce.com/products/list"
[
    {id: 1, name: "phone"},
    {id: 2, name: "computer"}, 
    {id: 2, name: "books"}   
]

// Create a schema
[{ id: "id", name: "text" }]

// mock response
[
    {id: 1, name: "lorem"},
    {id: 2, name: "ipsum"},
    {id: 3, name: "sit"},
]

Setup

Step 1. First clone project from github or clone git@github.com:sabbir-hossain/fake-server-local.git

Step 2. Run npm i (node.js required) to install dependency.

Step 3. Run npm start

Step 4. Now go to http://localhost:9920/dashboard. You should see the following image.

Create fake api

  • Create a new project. Now your api endpoint will be http://localhost:9920/${your-project-title}. you can create multiple project for multiple server.
  • select Route type from dropdown. Route type can be GET | POST | PUT | PATCH | DELETE.
  • create route. For example /some/api/name. So your first fake api api will be like http://localhost:9920/${your-project-name}${/some/api/name} and it will be Route type type.
  • create returned schema. It has to be a JSON object. key will be expected output response's key and value will be output response's type.
  • There are no save/update button. Once you move your cursor, value will be updated
  
// JSON object schema          | // Sample schema
{                              | {
  "key-01": "return-Type",     |  "id": "uuid",
  "key-02": ["return-Type"],   |  "name": {
  "key-03": {                  |    "first": "text",
    "child-01": "return-Type"  |    "last": "text"
  }                            |  }
}                              |  "hobbies": ["text"]
                               | }

More details

All you need to create a route and schema. Available schema's data type can be id, uuid, boolean, text, title, textarea, integer, float, phone, zipcode, date, time, date-time, url, email, image, pdf, csv, doc, ipaddress, second, alphanumeric.

id: incremental number (1, 2, 3, 4 etc)
uuid: uuid/v1
text: word (amet/fastidii/laboramus)
title: 2-5 words
textarea: 10-15 words
phone: US phone number
zipcode: number between 10001-99999
date: MM/dd/YYYY
time: HH:MM (AM/PM)
date-time: ISO Date String
second: Date in second
image/ pdf/ csv/ doc: link
alphanumeric: string with length of 25
boolean: true/false

We might have expected fixed value. For example, in json response, there might be a parameter, called message, that value will be always like message: "Successful". In this situation, simply add like { "key-name": "any-fixed-value" } if return-Type doesn't match with return-Type value, then expected value will be returned typed value.

We might have expected random value from fixed value list. For example, For example, in json response, there might be a parameter, called status hat value will be always be like status: "complete" or status: "failed". In this situation, simply add like { "object-key-name": "fixed-value-01,fixed-value-02" }. That means, if return-Type's value is comma-separated, then returned value will be one of comma-separated value.

We might have expected random array value. To get array value, try "object-key-name": ["return-Type"] (only one value and have to matched with return-Type). , see to bellow image.

In first image, It will return an array of ["return-Type"] and returned array contains about 1-10 results.

We create a special object, which contains "__type": "array" (means it return array value), "__range": "minRange,maxRange" return array data between __range. __range can also be fixed number, which means, it return fixed array length. "__property": "return-Type | object" array value will be based on __property. __property can be any return-Type or object

In third image, we add multiple array values, it will return a array, which contains one of array values.

We might want to expect random array object values. To get array object value, see to bellow image. In first image, we create an object array, and it will return 1-10 object array random values.

But sometimes, we want more data. In second image, we special object, which contains "__type": "array" (means it return array value), "__range": "minRange,maxRange" return array data between __range. __range can also be fixed number, which means, it return fixed array length. "__property": "return-Type | object" array value will be based on __property. __property can be any return-Type or object

We also need token data. see right images. Here we create a special object where __type is token and it required __property, __property can be object, array or return-Type

In POST | PUT | PATCH type api, if request.body's parameter match with user schema parameter, then request.body data will be returned
